# Contributing

Thank you for your interest in contributing to **awesome-ai-for-web3**.

## Guidelines

- **One pull request per addition.** Keep PRs focused.
- **Check for duplicates** before submitting a new entry.
- **Use the format:** `- [Name](URL) - Description ending with a period.`
  - Description should be concise (5–15 words), benefit-focused, and written in neutral tone.
  - Do not start descriptions with "A" or "An".
- **Place entries in the correct section.** If unsure, suggest a new section in your PR.
- **Only include actively maintained resources.** Check that the repo or service is not abandoned.
- **No self-promotion** without prior discussion in an issue.

## Entry Format Example

```
- [Protocol Name](https://github.com/org/repo) - Short description of what it does and why it is useful for AI agents.
```

## Sections

| Section | What belongs here |
|---|---|
| AI Agent Frameworks | Frameworks/platforms for building autonomous Web3 AI agents |
| Protocol-Specific AI Skills & Plugins | Installable skill modules for specific protocols |
| Web3 SDKs for Agents | SDKs for interacting with blockchains and DeFi protocols |
| DeFi APIs & Data | REST APIs for DeFi, token, and portfolio data |
| On-Chain Data & Indexing | Indexing protocols, block explorers, and query tools |
| Wallet & Account Abstraction | Smart account SDKs, ERC-4337 tools |
| Smart Contract Automation | Keepers, automators, and off-chain logic tools |
| RPC & Node Providers | Infrastructure and enhanced RPC services |
| MCP Servers for Web3 | Model Context Protocol servers exposing Web3 data/actions |
| AI Knowledge Bases & Education | Structured knowledge for teaching AI agents about Web3 |

## Pull Request Checklist

- [ ] Entry follows the format above
- [ ] URL is correct and accessible
- [ ] Resource is actively maintained
- [ ] No duplicate of an existing entry
- [ ] Added to the correct section in alphabetical order (where applicable)
